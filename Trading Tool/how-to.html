How to use Trading Tool

A weekly-or-monthly trend-following workflow ported from a Google Sheets screener. The tool ranks ~500 large-cap US stocks, tells you which ones meet the buy rules, and gives a regime call for sizing. You place the trades. It never does.

·

The 30-second pitch

If you read nothing else.

What it does

Pulls 14 months of daily prices for ~500 US large-caps, computes trend & momentum metrics, scores and ranks them, and renders a dashboard. Output: a list of names worth buying, holding, watching, or selling.

When to run it

Weekly or monthly. Monthly is simpler and reduces churn. Run on a Sunday evening before the trading week to plan adjustments.

How to trade it

Buy the top-ranked names flagged 1 ACTION BUY. Hold while the rank + trend stays favorable (1–6 months typical). Sell on rank decay or breach of the 50-day MA.

2

Command-line cheat sheet

All flags are optional. 
# Default run — top 500 by liquidity, opens dashboard.html in browser
python run.py

# Smaller universe (faster, but ranks against fewer names)
python run.py --top 200

# Include your own tickers from a file (one per line, # for comments)
python run.py --watchlist watchlist.txt

# Force-refresh prices (ignore the 15-min cache)
python run.py --fresh

# Build dashboard but don't auto-open browser (useful in cron)
python run.py --no-open

# Skip the sector-ETF screener (slightly faster)
python run.py --no-sectors

# Export a Pine Script alert file for the session's top 25 ACTION BUYs
# → writes dashboard_pine_alerts.pine next to run.py
python run.py --export-pine

Output files land next to run.py: dashboard.html (the page) and dashboard.csv (the raw ranked table — drop into Excel or pandas for your own analysis).

The 8-step action ladder

Every ticker ends in exactly one of these buckets. Filter the dashboard table by bucket.
1 BUYACTION BUYAbove 200D · Signal BUY · entry A or B · → take a starter or full position
2 STARTERStarter / Scale-InAbove 200D · BUY · entry C (acceptable but a bit extended) · half-size
3 WATCH+Buyable WatchAbove 200D · Signal HOLD · entry A/B/C · already holding? keep; not yet? wait for setup
4 WATCHSecondary WatchAbove 200D · Signal WATCH · entry A/B/C · interesting but rank not high enough
5 RECLAIMWait for ReclaimBelow 9D average · wait for it to reclaim 9D before re-entering
6 STALKStalk PullbackStrong name but too extended (entry E/F/G) · wait for a healthy pullback
7 LOWLow PrioritySELL signal but clean entry · de-prioritize, watch trend
8 PASSPassNothing actionable · skip

Triage tip. Each week scan the 1/2/3 buckets first. If your portfolio holds a name now in 5/6/7/8, that's the sell candidate.

6

The TradingView Trigger Layer

Five confirmation signals layered on top of the trend-following rank.

The rank tells you which names are leading. The trigger layer tells you when a leader is firing a confirmation pattern that aligns with what top quant systems watch. Each of these is computed from the OHLCV data already in the cache — no new API calls, no new deps. A name with TV Count ≥ 2 is showing multiple confirmations at once.

🔥 Squeeze

squeeze_fire

The TTM Squeeze just released bullishly: Bollinger Bands (20, 2σ) were inside Keltner Channels (20-EMA ± 1.5·ATR) on the prior bar, are now outside, and the 20-bar momentum histogram is positive.

Best when: combined with rank ≤ 20 and Above 200D.
⏳ Squeeze On

squeeze_on

Bollinger Bands are currently inside Keltner Channels — volatility compression. A coil before a potential expansion.

Best when: stalk-list candidate. Wait for squeeze_fire before entering.
📈 MACD

macd_bull_cross

MACD histogram (12 EMA − 26 EMA, signal = 9 EMA of MACD) crossed from negative to positive in the last 3 bars, with price still above the 200-day MA.

Best when: a name you already wanted to buy is also flashing this.
↗ RSI Div

rsi_bull_divergence

Hidden bullish divergence in the last 30 bars: price made a higher low while RSI(14) made a lower low. Continuation pattern in established uptrends.

Best when: pullback / Entry Grade D scenarios. Hints the dip is being defended.
📊 Vol Surge

vol_surge

Today's volume is > 1.5× the 20-bar average and the close is above the prior day's high. A breakout bar with confirming volume.

Best when: aligns with a 1 ACTION BUY or 3 BUYABLE WATCH on the same name.

How to use them. The triggers are confirmations, not standalone buy signals. Filter the scanner by 1 BUY first, then by a trigger pill (e.g. 📈 MACD Cross or ⚡ Any Signal) to see which top-ranked names are firing right now.

7

Pine Script export → TradingView alerts

Take the trigger layer with you onto a real TradingView chart.

--export-pine writes a self-contained Pine Script v6 indicator (dashboard_pine_alerts.pine) that reproduces all five triggers natively in TradingView. Drop it into the Pine Editor, add it to a chart, and wire up webhook alerts. 

# 1. Run the screener and export the Pine file for the top 25 ACTION BUYs
python run.py --top 200 --export-pine

# 2. Open dashboard_pine_alerts.pine, copy its full contents.
# 3. In TradingView: Pine Editor → paste → "Add to chart".
# 4. Right-click chart → Add alert → Condition: "TV Trigger Layer"
#    → pick Squeeze Fire / MACD / RSI Div / Vol Surge / Any Trigger.
# 5. Set Options to "Once Per Bar Close" and paste your webhook URL.
# 6. Repeat per symbol (TradingView fires alerts per indicator instance).

The generated alert messages already include {{ticker}} and {{close}} placeholders, so your webhook receiver (Discord, Slack, n8n, etc.) gets a ready-to-route payload like 🔥 NVDA squeeze FIRED bullish @ 950.12.

Glossary

The columns & metrics, plain English.
Leadership ScoreLong-term momentum composite: 3M·0.3 + 6M·0.25 + 12M·0.1 + RS·0.2 + min(20D above, 30%)·0.05 + min(5D, 15%)·0.1. Higher = stronger trend.
Momentum ScoreShort-term composite: 20D above·0.6 + 5D return·0.4. How fresh is the move?
Blended ScoreRank-weighted blend: LeadRank·0.4 + RS20Rank·0.25 + MomRank·0.35. Lower = better (it's ranks, not scores).
Final ScoreBlended·0.75 + LocationScore·0.25, then ×0.7 if extension > 3. The number that determines your Final Rank.
Raw ScoreA simpler alternate: 3M·0.35 + 6M·0.35 + 12M·0.2 + RS·0.1. No entry quality. From the Excel "Rankings" tab.
Extension Ratio(price − 9D MA) / ATR. How many ATRs above the 9-day average. >3 = parabolic, hit a 0.7× penalty on Final Score.
ATR (14)14-bar Average True Range — volatility in dollar terms. Used to normalize the Extension Ratio.
Entry GradeA (ideal, ext≤0.5) → B (0.5–1) → C (1–1.5) → Watch (1.5–2) → Extended (2–3) → Chase (>3) → Below 9D (pullback).
Rule CheckAbove 200D · Above 50D only · Below 50D. The trend filter. Only names "Above 200D" can flag as 1 ACTION BUY.
RS vs SPY 3MTicker's 3-month return minus SPY's 3-month return. Positive = outperforming the market.
TV SignalsColored pills naming whichever of the 5 TradingView triggers are firing on the latest bar. See section 6.
TV CountInteger 0–5 — how many triggers are firing at once. Sortable; ≥ 2 means multiple confirmations.
squeeze_on / fireTTM-style volatility compression detector. on = coiling; fire = just released with positive momentum.
macd_bull_crossMACD histogram crossed from negative to positive in the last 3 bars while price is still above the 200D MA.
rsi_bull_divergenceHidden bullish divergence: price made a higher low, RSI(14) made a lower low, within the last 30 bars.
vol_surgeToday's volume > 1.5× the 20-bar average and close above the prior day's high. A confirmed breakout bar.

The rules (from the original Process tab)

The framework, verbatim.

Review cadence

Weekly or monthly; monthly is simpler and reduces churn.

Buy rule

Top rank, price > 50D, price > 200D, regime not Defensive.

Hold rule

Rank ≤ 12 and price > 50D.

Watch rule

Rank ≤ 20 and price within ~2% of the 50D average.

Sell rule

Weak rank or price < 50D.

Defensive posture

Regime Defensive → idle capital in BIL/SGOV or money-market funds.

Sizing by regime

Risk On 5–8 names · Mixed 3–5 · Defensive 0–2.

Holding period

1–6 months typical; sometimes longer. Hold while rank + trend stay favorable.

Mindset

Buy strength, not cheapness. Follow leadership, not bottoms.

If it breaks

Quick diagnostics for common issues.

"Failed to fetch SPY"

Network issue, Yahoo rate-limit, or VPN blocking. Retry; if persistent, check yfinance upstream issues.

"possibly delisted: TICK"

Ticker was acquired/spun-off. Harmless — engine skips with a warn. Add the ticker to universe.py's removed list to silence it.

Dashboard shows zero BUYs

Either regime is Defensive (no BUYs by design), or every top-ranked name is extended. Re-check next session.

Numbers look stale

Cache is 15 min. Run python run.py --fresh to force a re-download.

"No 1 ACTION BUY tickers — skipping Pine export"

Expected in Defensive regimes or when nothing meets the BUY rule. Re-run when conditions improve, or lower --top to widen the universe.

TV column shows "—" everywhere

Universe is < 30 bars deep (new tickers in your watchlist?). Triggers default to False when there isn't enough history. Wait a few sessions.
